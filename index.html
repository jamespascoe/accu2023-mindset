MINDSET

Mindset

  • noun. a way of thinking
    •
  • C++ has lots of mindsets:
    •
    • Imperative
      •
    • Functional
      •
    • Concurrent
      •
    • Object-Oriented
      •
    • Compile-time
      •
    • ...
      •
  • Blending these together leads to mastery
    •

Imperative Mindset

  • Solve problems by expressing each step
    •
  • Lots of advantages:
    •
      ... easy to read, learn and switch
  • Optimise code for readability
    •
  • Ref: C++ Core Guidelines

Functional Mindset

  • Evaluate expressions (rather than statements)
    •
  • Expressions consist of functions
    •
  • Declarative: the programmer states what needs to be done and the language determines how
    •
  • Ref: Functional Programming in C++ (Ivan Čukić)
It is a declarative programming paradigm in which function definitions are trees of expressions that map values to other values, rather than a sequence of imperative statements which update the running state of the program.

Concurrent Mindset

  • Multiple work items 'in progress' at the same time
    •
  • Parallelism: items are processed at the same instant
    •
    • ... often implies extra hardware: cores, ILP etc.
  • Concurrency design is important
    •
    • Eliminate bugs by design (e.g. race conditions)
  • Be careful when switching mindset
    •
  • Ref: Concurrency in Action (Antony Williams)

Conclusions

  • C++ consists of many mindsets
    •
    • imperative, concurrent, functional ...
  • Study each one in isolation
    •
  • Recognise when to change mindset
    •
  • Blending them together leads to mastery
    •
